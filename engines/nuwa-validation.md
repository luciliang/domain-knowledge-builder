# 心智模型验证规范（领域知识库）

> Pipeline S5（心智模型提炼）加载本规范。它把 nuwa 蒸馏「一个人怎么想」的三重验证方法论，
> 迁移为提炼「一个领域怎么思考」的标准。心智模型是领域的镜片，不是某人的口音。

---

## 一、为什么需要心智模型验证

领域知识库有两种写法，质量天差地别：

**纯定义堆砌**——把定义、定理、方法逐条搬进来，能查不能想。问"什么是分割共形预测"，它能回答；
问"我的数据流是非交换的，这个领域会怎么帮我"，它答不出，因为它只存了名词解释，没存思维方式。

**心智模型驱动**——先提炼这个领域用什么镜片看世界，再用知识节点去支撑镜片。同一个问题，
它能给出有立场的推断：例如共形预测（CP）的核心镜片是"只要校准分数与测试分数可交换，覆盖率就无条件成立"。
握住这条镜片，面对任何新数据类型，领域的第一反应都是"先问它能否满足可交换性"，而非翻字典找现成方法。

心智模型的检验标准不是"听起来深刻"，而是"能否运行"：
它必须能解释已有结论、能预测对新问题的立场、能区分本领域与相邻领域的思维差异。
达不到这三点，所谓"模型"只是漂亮的概括句，没有生成力。

本规范就是把"能否运行"拆成可执行的三重验证，并在领域语境下定义矛盾处理、信息不足处理与质量自检。

**同一个难题，两种知识库给出不同的回答**——

难题：「我的在线推荐系统数据持续漂移，CP 还能用吗？」

- 纯定义堆砌型：只能罗列"分割 CP 定义""加权 CP 定义"，把拼图扔给用户。
- 心智模型驱动型：直接落到镜片上——"漂移破坏了校准集与测试集的可交换性，这是 CP 合法性的根基。先估密度比做加权校正恢复可交换性，否则覆盖率保证直接失效。"

后者一步到位，是因为它持有领域的思维镜片，而不只是一个名词索引。

## 二、三重验证（心智模型识别）

一个提炼出的论点要被认定为「心智模型」而非「随手概括」，必须通过三重验证。

### 验证1：跨场景复现

同一个思维框架出现在领域的至少 2 个不同子问题/子领域/来源论文中。
单独出现一次的，可能只是某篇论文针对特定问题的技巧，不足以称模型。

例（共形预测）：镜片「可交换性即合法性」——
- 在标准分割 CP 中：可交换 → 覆盖率 $\geq 1-\alpha$ 无条件成立
- 在协变量漂移下：可交换被打破 → 用密度比加权恢复（加权 CP）
- 在结构化数据中：用更弱的对称排列不变性（SymmPI）替代可交换性
→ 跨 3 个子设定复现 → 这是真正的心智模型，不是某一篇的局部结论

> 判定信号：把镜片表述写下来，去节点库里数它能在几个独立子问题/来源里找到支撑节点。≥2 个不同子问题才算复现；只在同一篇论文的同一节反复出现的不算。

### 验证2：有生成力

用这个模型可以推断领域对一个**新问题**的可能立场与方法选择。能生成新推断，才算运行得起来的模型。

例（共形预测）：若「可交换性即合法性」是心智模型，那么面对新问题"流数据/时序上的覆盖保证"——
- 领域的第一反应不是"找现成方法"，而是先问"时序破坏了可交换性吗？"
- 若破坏 → 倾向加权 / 在线校正路线；若可恢复 → 标准 CP 即可
→ 能生成对新问题的立场推断 → 这是真正的心智模型

> 判定信号：临时编一个文献里没有直接讨论的新问题，问"用这个镜片能推出领域会怎么应对吗"。推得出且有立场 → 有生成力；只能复读原文已有结论 → 没有。

### 验证3：有排他性（领域独特视角）

这个思维框架是**本领域特有**的，换一个相邻领域就不成立或不必要。
没有排他性的"模型"，往往只是常识性逻辑，提炼了等于没提炼。

例（共形预测）：「可交换性」这条镜片是 CP 特有的——
- 贝叶斯方法不需要它：贝叶斯靠"先验正确"获得保证
- 频率派大样本方法不需要它：靠渐近理论
- 只有 CP 把"可交换性"当作合法性本身来使用
→ 换到贝叶斯领域这条镜片就不成立 → 有领域区分度 → 值得提炼

反例（不通过排他性）：「相关不等于因果」是几乎所有量化领域共有的常识，
没有体现 CP 特有的视角，即便出现在文献里，也不算 CP 的心智模型。

> 判定信号：把这个镜片套到一个相邻领域（如把 CP 的可交换性套到贝叶斯）——它立刻失效或不必要 → 有排他性；套到哪个领域都成立 → 只是常识，降级或丢弃。

### 降级规则

| 通过验证数 | 归类 | 处理 |
|------------|------|------|
| 3 重全过 | 心智模型 | 纳入，作为镜片写入 SKILL.md |
| 通过 1-2 重 | 决策启发式 | 降级，作为"在该场景下怎么做"的实操规则保留 |
| 0 重 | 不纳入 | 可能只是某篇论文在特定场景的一句话，丢弃 |

降级例：提炼出"优先选 APS 而非 RAPS"——它只在分类预测集这一处出现（验证1 不通过），
虽好用但无跨场景复现、不构成思维镜片 → 降为决策启发式，不冒充心智模型。

**一个完整的分级示例**（同为 CP 领域的三个候选）：

| 候选镜片 | 跨场景 | 生成力 | 排他性 | 归类 |
|----------|:------:|:------:|:------:|------|
| 可交换性即合法性 | ✓×3 | ✓ | ✓ | 心智模型（3 重全过）|
| 优先用 APS 而非 RAPS | ✗ | ✓ | ✗ | 决策启发式（过 1 重）|
| 校准集越大区间越窄 | ✓ | ✗ | ✗ | 丢弃（常识，非领域镜片）|

三者看起来都"对"，但只有第一个是值得写进 SKILL.md 顶部的镜片。

> 领域版的锚点迁移：原文锚定的是「某位学者 X 的独特视角」；
> 这里锚定的是「领域的核心张力/独特镜片」。判断标准从"换个人就不这么想"
> 变成"换个领域就不这么想"。

## 三、矛盾处理（领域内不同流派的分歧）

分歧是领域的活力来源，不是需要修复的 Bug。
一个只有共识、没有张力的知识库，说明调研还不够深。

### 三种分歧类型

1. **时间性分歧**（观点演化）
   - 早期文献主张 A，近期文献主张 B
   - 处理：记录演化轨迹，标注「早期」「近期」，不把旧结论改写成新结论
   - 例：早期 CP 文献聚焦边际覆盖，近期转向条件覆盖的不可能性与三误差分解

2. **设定性分歧**（不同应用场景，不同规则）
   - 同一领域在不同应用设定下给出不同方法选择
   - 处理：分场景记录，不强求统一为一条规则
   - 例：医学影像要语义分组（按器官）覆盖，一般回归只要边际覆盖——两者都对，分场景适用

3. **本质性张力**（领域内在的结构性冲突）
   - 两个目标在本领域内原则上不可同时满足
   - 处理：明确记录为「核心张力」，并指出它在理论上的不可调和之处
   - 例：边际覆盖廉价可达，条件覆盖在一般情况下不可能（已证明）——这通常是最有洞察的部分

### 错误的处理方式

- ❌ 选一边，忽略另一边（制造虚假共识）
- ❌ 编一个调和的解释，假装矛盾被解决了
- ❌ 假装分歧不存在

正确做法：在心智模型/节点中并列呈现分歧双方，标注各自的适用边界与来源，
让使用者自己判断在具体问题上取哪一边。

**如何在知识库中记录一个本质张力**（示例格式）：

> **核心张力：边际覆盖 vs 条件覆盖**
> - 一方：边际覆盖廉价、有限样本可达、分布自由（`thm-split-cp-coverage`）
> - 另一方：条件覆盖在一般情形已被证明不可能（`thm-conditional-coverage-impossible`）
> - 不可调和之处：不是工程问题，是理论上的不可能，只能追求近似
> - 来源：Vovk 2012；Min et al. 2026 三误差分解给出精确诊断

并列呈现双方 + 点明不可调和点 + 挂上节点 ID，使用者就能判断自己的问题落在张力哪一侧。

## 四、信息不足处理（领域覆盖不全时的诚实标注）

领域知识库的来源总是有限的，覆盖不全时必须诚实，不能补脑。

| 情况 | 处理 |
|------|------|
| 某子领域文献覆盖很少 | 在 SKILL.md 标注「知识不足，此方向为推测」，不假装精通 |
| 只有单一来源支撑某结论 | 降低置信度，标注「仅据[来源]」，提醒未经交叉验证 |
| 来源间结论冲突且无法判断 | 并列呈现，列出各方适用条件，让使用者自行判断 |
| 该方向是领域开放问题，尚无共识 | 尊重边界，标注「此为开放问题，领域尚无定论」，不强行给答案 |

诚实标注不是减分项：明确写出"做不到什么"的知识库，比假装无所不知的更可信、更可用。

**置信度标注的实操**：

- 单一来源支撑：「据[Angelopoulos & Bates 2022]，分割 CP 在可交换下有限样本有效」——加「未经第二来源交叉验证」标注，使用者据此降权。
- 推测性结论：从已有理论外推但无直接文献支撑时，写「此为基于可交换性镜片的推测，尚无文献直接验证」，而非当成定论陈述。
- 开放问题：「条件覆盖的最优近似度量（FSC vs SSC）领域尚无统一标准」——如实留作开放，不替领域拍板。

**何时该拒绝回答**：当问题落在明确标注的未覆盖子领域、且没有可靠镜片可外推时，诚实地说"本知识库未覆盖此方向，无法给出有依据的回答"，远胜于用定义堆砌拼一个看似合理的答案。承认边界本身就是领域素养的一部分。

## 五、质量自检清单

pipeline S5 产出后，逐项自检。任一不通过，回到验证循环。

### 心智模型
- [ ] 每个模型都有来自 **≥2 篇来源 / 子领域** 的跨源证据？（领域版：跨源，不只是跨段落）
- [ ] 模型数量在 **3-7 个** 之间？（太少=太浅，没提炼；太多=没收敛，全是概括句）
- [ ] 每个模型都有明确的应用场景**和**局限边界？
- [ ] 模型之间可以有张力，但不互相矛盾到无法并存？

### 决策启发式
- [ ] 每条规则都有具体案例 / 节点 ID 支撑？
- [ ] 可以被**新情况**触发，而非只适用于提炼时的原始案例？

### 诚实边界
- [ ] 明确写了这个知识库做不到什么、未覆盖哪些子领域？
- [ ] 标注了来源论文清单和调研时间？
- [ ] 承认了覆盖不足或信息不足的维度？

### 整体（领域版"换皮测试"）
- [ ] 用这个领域的镜片看一个**新问题**，能得到有价值的、有立场的推断？
- [ ] 读起来是框架在运行，而不是论文摘抄的拼凑？
- [ ] **删掉领域名称后，还能认出这是哪个领域的思维方式？**（排他性的终极自检）
  - 能认出 → 真领域镜片；谁都能用 → 只是常识，重提

## 六、Pipeline S5 如何调用本规范

S5（心智模型提炼）是 DAG 流水线的收口步骤，把散落的节点蒸馏成可运行的镜片。

### 输入

- 所有 knowledge 节点：definitions / theorems / methods / insights / experiments
- sources 摘要：每篇来源论文的核心贡献与覆盖范围

### 处理流程

1. **候选识别**——扫描定理与 insight 节点，提炼候选"镜片"表述（不是定义，是看世界的方式）
2. **三重验证**——对每个候选逐项跑验证1/2/3，收集跨源证据：
   - 跨场景：该镜片在几个子问题/来源里复现？列出对应节点 ID
   - 生成力：假设一个新问题，这个镜片能预测领域的什么反应？
   - 排他性：换个相邻领域，这条镜片还成立吗？
3. **分级**——按降级规则归类：3 重过=心智模型，1-2 重=决策启发式，0 重=丢弃
4. **矛盾标注**——把领域内分歧按三类（时间性/设定性/本质张力）挂到相应模型上
5. **诚实边界**——汇总覆盖盲区、单一来源结论、开放问题

### 输出

- **3-7 个验证过的心智模型**：每个含跨源证据节点 ID、应用场景、局限、（若有）核心张力
- **5-10 条决策启发式**：从只过 1-2 重的候选中降级保留的实操规则，每条带触发场景与案例节点
- **诚实边界段落**：未覆盖子领域 + 单一来源结论 + 开放问题 + 来源清单 + 调研时间

### 一次完整运行样例

输入：CP 知识库 50 个节点 + 3 篇来源摘要。

- **候选识别**：扫到 7 个候选镜片（可交换性、边际/条件张力、风险泛化、语义锚点、大校准集、APS 选型、相关≠因果……）
- **三重验证 + 分级后**：
  - 心智模型（3-7 个）：可交换性即合法性 / 边际-条件张力 / 风险泛化器 / 不确定性要有语义锚点（共 4 个，全过 3 重）
  - 决策启发式（5-10 条）：优先 APS 选型、校准集与测试集同分布、分数设计优先于算法调参……（过 1-2 重降级而来）
  - 丢弃："相关≠因果"（0 重，常识）、"校准集越大区间越窄"（常识）
- **诚实边界**：标注未覆盖交叉共形、时序/流数据、对抗数据等子领域 + 单一来源结论 + 开放问题

自检条数：心智模型 <3 → 回去扩大候选来源（多半来源太少）；>7 → 回去合并/丢弃（没收敛）。这是数量信号，不是凑数。

### 与查询协议的衔接

产出的心智模型写入 SKILL.md 的「核心心智模型」段，成为 DAG 查询时综合回答的镜片：
加载 3-5 个节点后，不是照搬节点内容，而是**用镜片组织节点、推断立场、指明局限**。
若一个候选提炼后无法支撑这种"镜片式回答"，说明它没有真正通过三重验证，应降级或丢弃。
