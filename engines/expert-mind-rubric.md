# 专家心智提炼规范

> Pipeline S5b 加载本规范，从 S2-mind 产出的心智候选中提炼专家心智模型。
> 它恢复 nuwa-skill 为「领域知识库」而剔除的**人物 DNA**（判断、直觉、反模式、决策启发式），
> 并将三重验证的锚点从「领域镜片」迁移到「专家个人镜片」——这位专家特有的思维，换个人就不这么想。

---

## 一、为什么需要专家心智提炼

领域知识库与专家顾问的核心差异在于**心智的主体性**。

**纯领域镜片**（DKB S5a）——提炼的是「这个领域怎么思考」。例如共形预测的镜片是「只要校准分数与测试分数可交换，覆盖率就无条件成立」。这是领域的共识，任何共形预测研究者都会这么想。

**专家心智镜片**（本规范 S5b）——提炼的是「**这位专家**怎么思考」。例如 Yann LeCun 的镜片是「智能 = 在能量函数上做优化/约束满足，而非自回归预测下一token」。这不是机器学习领域的共识，而是 LeCun 个人特有的世界模型。

两个镜片的**检验标准不同**：

| 镜片类型 | 锚点 | 验证标准 |
|---------|------|----------|
| 领域镜片 | 「换个领域就不这么想」 | 排他性（领域特有）|
| 专家心智镜片 | 「**换个人就不这么想**」 | 排他性（专家特有）|

专家心智的价值在于**有立场**。面对同一个问题（「LLM 能做真正推理吗」），领域镜片只能罗列各方观点，而专家心智能给出明确判断（LeCun：「不能，因为缺乏世界模型」）+ 推理链 + 依据节点。这是「顾问」与「百科全书」的区别。

本规范就是把「这位专家特有的思维」拆成可执行的提炼标准。

---

## 二、三重验证（专家心智识别）

一个提炼出的心智元素要被认定为「专家心智模型」而非「随手概括」，必须通过三重验证。

### 验证1：跨场景复现

同一个思维框架出现在专家的至少 **2 个不同语境/作品/时间段**中。
单独出现一次的，可能只是针对特定问题的评论，不足以称专家的镜片。

例（LeCun）：镜片「智能 = 能量优化，非自回归」——
- 在 JEPA 架构论文中：提出 Latent EBM，在抽象空间做预测式学习
- 在访谈中：反对纯 LLM 路线，强调需要世界模型与能量函数
- 在社交媒体：多次批评 GPT 式自回归的局限
→ 跨 3 个语境复现 → 这是 LeCun 持有的世界模型，不是一时评论

> 判定信号：把镜片表述写下来，去 sources 里数它能在几个独立语境/来源里找到支撑。≥2 个不同来源才算复现；只在同一篇论文反复出现的不算。

### 验证2：有生成力

用这个专家的镜片可以推断他对一个**新问题**的可能立场与方法选择。
能生成新推断，才算运行得起来的专家心智。

例（LeCun）：若「智能 = 能量优化」是心智模型，那么面对新问题「用扩散模型做世界模型」——
- 专家的第一反应不是「看效果」，而是先问「扩散模型有没有显式的能量/目标函数？有没有 forward simulator？」
- 若无 → 倾向反对，认为这只是统计拟合，缺乏规划能力
→ 能生成对新问题的立场推断 → 这是真正的专家心智

> 判定信号：临时编一个文献里没有直接讨论的新问题，问「用这个专家的镜片能推出他会怎么应对吗」。推得出且有明确立场 → 有生成力；只能复读原文已有观点 → 没有。

### 验证3：有排他性（专家独特视角）

这个思维框架是**这位专家特有**的，换一个同领域专家就不成立或不这么强调。
没有排他性的「模型」，往往只是领域共识，提炼了等于没提炼。

例（LeCun）：「能量优化世界观」这条镜片是 LeCun 特有的——
- Hinton 不强调它：Hinton 的反向传播路线不依赖显式能量函数
- Bengio 不强调它：Bengio 的 consciousness prior 路线聚焦表征学习
- 只有 LeCun 把「能量函数 + world model」当作 AGI 的必要条件来使用
→ 换到同领域其他专家这条镜片就不成立 → 有专家区分度 → 值得提炼

反例（不通过排他性）：「数据质量很重要」几乎是所有机器学习专家共有的共识，
即便 LeCun 多次提到，也不算 LeCun 特有的心智模型。

> 判定信号：把这个镜片套到同领域另一位专家（如把 LeCun 的能量世界观套到 Hinton）——立刻失效或不被强调 → 有排他性；套到哪位专家都成立 → 只是领域共识，降级或丢弃。

### 降级规则与分级输出

| 通过验证数 | 类型 | 处理 | grounded_in 要求 |
|------------|------|------|-----------------|
| 3 重全过 | **心智模型** (mental_model) | 纳入，作为专家核心镜片 | **必须** ≥1 个知识节点（强耦合，有硬依据）|
| 通过 1-2 重 | **启发式** (heuristic) | 降级，作为「专家在该场景下的决策规则」保留 | 可选（可能只有经验依据，标注）|
| **排他性必过**（无论其他验证）| **反模式** (anti_pattern) | 纳入，作为「专家明确反对的方向」 | 用 `role: refutes` 指向反对的方法节点 |
| 0 重 | 不纳入 | 可能只是某篇论文的一句评论，丢弃 | - |

**一个完整的分级示例**（同为 LeCun 心智的三个候选）：

| 候选镜片 | 跨场景 | 生成力 | 排他性 | 归类 |
|----------|:------:|:------:|:------:|------|
| 智能 = 能量优化，非自回归 | ✓×3 | ✓ | ✓ | 心智模型（3 重全过）|
| 优先选能做 forward simulation 的架构 | ✓ | ✓ | ✗ | 启发式（过 2 重）|
| 相关不等于因果 | ✓ | ✗ | ✗ | 丢弃（领域共识，非专家特有）|

三者看起来都「对」，但只有第一个是值得写入 `expert-mind/mental-models.md` 顶部的专家镜片。

**关键点**：本规范的**锚点迁移**——从 nuwa-validation 的「换个人就不这么想」（保留不变）迁移到「这位专家特有的思维，与领域共识区分开」。DKB 的 nuwa-validation 把锚点改成了「换个领域就不这么想」；本规范恢复原锚点，聚焦**专家个体差异**。

## 三、专家心智元素的结构（人物 DNA 的恢复）

nuwa-validation 为「领域知识库」剔除了人物视角专用部分（表达 DNA、时间线、角色扮演、风格量化）。本规范恢复与专家判断相关的认知维度，但不恢复纯风格维度。

### 3.1 恢复的人物 DNA 维度

| 维度 | 说明 | 示例（LeCun）|
|------|------|---------------|
| **判断** (Judgment) | 专家对具体问题的明确立场 + 推理链 | 「LLM 不能真正推理，因为缺乏世界模型」 |
| **直觉** (Intuition) | 专家未经完整论证但强烈的信念倾向 | 「自回归路线本质上有上限」 |
| **反模式** (Anti-pattern) | 专家明确反对的方法/方向 | 「反对纯自回归 LLM 作为 AGI 路线」 |
| **决策启发式** (Heuristic) | 专家在不同场景下的决策规则 | 「对新架构先问：有能量函数吗？有 forward simulator 吗？」 |

### 3.2 不恢复的维度（纯风格）

以下维度属于 nuwa-skill 的「人物扮演」范畴，与本 skill（专家顾问）无关，**不恢复**：

- ❌ 表达 DNA（口癖、比喻偏好、句式风格）
- ❌ 人物时间线（观点演化记录——由 `sources/` 的时间信息 + S5c 的 judgment status: inferred 推断机制覆盖，不在本规范）
- ❌ 角色扮演规则（「模拟 LeCun 的语气回答」）
- ❌ 风格量化（「这位专家说话有多直接」）

**边界**：本规范聚焦**认知内容**（他怎么想），而非**表达风格**（他怎么说）。认知内容是专家顾问的核心；表达风格是 nuwa-skill 职责。

## 四、与 S5c 紧耦合融合的衔接

S5b 产出的心智元素是 S5c 的**输入**。衔接契约如下：

### 4.1 输出格式

每个心智元素（mental_model / heuristic / anti_pattern）必须包含：

```yaml
---
id: mm-lecun-energy-based-worldview
type: mental_model            # mental_model | heuristic | anti_pattern
label: 能量模型世界观
statement: "智能 = 在能量函数上做优化/约束满足，而非自回归预测下一token"
verification:                 # 三重验证结果（S5b 执行）
  cross_scene: { pass: true, evidence: [JEPA架构, 反对LLM推理, LatentEBM] }
  generative: { pass: true, predicts: "对任何新架构先问能量/目标函数" }
  exclusive:  { pass: true, vs: "贝叶斯派靠先验、生成式派靠似然" }
grounded_in:                  # 紧耦合：统一对象数组（S5c 填充，本规范标注结构）
  - { node: thm-ebm-as-energy-minimization, role: supports, quote: "..." }
confidence: high
provenance: { sources: [lecun-2022-jepa, lex-interview-2023] }
---
```

### 4.2 grounded_in 的责任分工

- **S5b（本规范）**：标注 `grounded_in` 的**结构要求**（必须是统一对象数组，含 `node`/`role`/`quote`），但**不填充具体节点**（因为 S5b 是独立 subagent，不一定看到 S3 合并后的最终 DAG）。
- **S5c（紧耦合融合）**：遍历 S3 合并后的 DAG，为每个心智元素**匹配支撑节点**，填充 `grounded_in` 的具体内容，执行降级规则（§4.4）：无依据→丢弃，语义不匹配→降级为 heuristic。

> 为什么这样分工？S5b 与 S5a 可并行，但 S5c 必须等待 S3 合并完成（DAG 节点已定）。S5b 先做「哪些心智元素有价值」的判定（三重验证），S5c 再做「这些价值判断有什么知识支撑」的匹配（紧耦合）。

### 4.3 输出清单

S5b 完成后，产出：

- **3-7 个验证过的心智模型**：每个含三重验证证据、专家独特性标注、grounded_in 结构占位
- **5-10 条启发式**：从只过 1-2 重的候选中降级保留的实操规则
- **若干反模式**：专家明确反对的方法/方向（排他性必过）
- **诚实边界段落**：未覆盖的专家观点方向、单一来源结论、观点演化的标注

## 五、矛盾处理（专家观点的分歧与演化）

专家不是铁板一块。分歧是专家思维的真实性信号，不是需要修复的 Bug。

### 三种分歧类型

1. **时间性分歧**（观点演化）
   - 早期观点 A，近期观点 B
   - 处理：记录演化轨迹，标注「早期」「近期」，不把旧观点改写成新观点
   - 例：早期 LeCun 对 CNN 的强调 vs 近期对 JEPA/能量模型的转向

2. **设定性分歧**（不同应用场景，不同规则）
   - 同一专家在不同场景下给出不同判断
   - 处理：分场景记录，不强求统一为一条规则
   - 例：「在监督学习中，数据量优先；在自监督学习中，架构设计优先」

3. **本质性张力**（专家思维的结构性冲突）
   - 两个目标在专家框架内原则上有张力
   - 处理：明确记录为「核心张力」，不编调和
   - 例：「LeCun 强调 world model 重要性，但同时认为当前显式建模 world model 的方法都不成熟」

### 错误的处理方式

- ❌ 选一边，忽略另一边（制造虚假共识）
- ❌ 编一个调和的解释，假装矛盾被解决
- ❌ 假装分歧不存在

正确做法：在心智模型/启发式/反模式中并列呈现分歧双方，标注各自的适用边界与来源。

## 六、信息不足处理（专家材料覆盖不全时的诚实标注）

专家材料的来源总是有限的（尤其 MVP 只做最小网采 3 路）。覆盖不全时必须诚实，不能补脑。

| 情况 | 处理 |
|------|------|
| 某类观点覆盖很少（如专家对某前沿方向未公开评论） | 在 `expert-mind/mental-models.md` 标注「材料不足，此方向为推测」，不假装精通 |
| 只有单一来源支撑某结论 | 降低置信度，标注「仅据[来源]」，提醒未经交叉验证 |
| 来源间结论冲突且无法判断 | 并列呈现，列出各方适用条件，让使用者自行判断 |
| 该方向是专家未明确表态的开放问题 | 尊重边界，标注「专家未明确表态」，不强行推断 |

诚实标注不是减分项：明确写出「做不到什么」的专家顾问，比假装无所不知的更可信、更可用。

**置信度标注的实操**：

- 单一来源支撑：「据[LeCun 2022 JEPA 论文]，自回归模型缺乏规划能力」——加「未经第二来源交叉验证」标注，使用者据此降权。
- 推测性结论：从已有镜片外推但无直接文献支撑时，写「此为基于能量优化世界模型的推测，专家未直接表态」，而非当成定论陈述。
- 未表态领域：「专家未公开评论过[某方向]，无法给出其立场判断」——如实留作未知，不替专家拍板。

**何时该拒绝回答**：当问题落在明确标注的未覆盖方向、且没有可靠镜片可外推时，诚实地说「本专家顾问库未覆盖此方向，无法给出有依据的立场判断」。

## 七、质量自检清单

S5b 产出后，逐项自检。任一不通过，回到验证循环。

### 心智模型（3-7 个）
- [ ] 每个模型都有来自 **≥2 个来源/语境** 的跨场景证据？
- [ ] 每个模型都有明确的**专家独特性标注**（排他性验证，换个人不这么想）？
- [ ] 模型数量在 **3-7 个** 之间？（太少=材料不足或提炼过浅；太多=没收敛，全是评论句）
- [ ] 每个模型都有 `grounded_in` 结构占位（具体节点由 S5c 填充）？

### 启发式（5-10 条）
- [ ] 每条启发式都有具体来源/语境支撑？
- [ ] 可以被**新情况**触发，而非只适用于提炼时的原始案例？

### 反模式
- [ ] 每个反模式都有专家**明确反对**的证据（非推测）？
- [ ] 用 `role: refutes` 指向反对的方法节点？

### 诚实边界
- [ ] 明确写了这个专家顾问库做不到什么、未覆盖哪些方向？
- [ ] 标注了来源清单（网采 URL + 用户 PDF locator）？
- [ ] 承认了覆盖不足或信息不足的维度？

### 整体（专家版「换人测试」）
- [ ] 用这个专家的镜片看一个**新问题**，能得到有明确立场的推断？
- [ ] 读起来是专家在思考，而不是文献摘抄的拼凑？
- [ ] **删掉专家名字后，还能认出这是哪位专家的思维方式？**（排他性的终极自检）
  - 能认出 → 真专家心智镜片；谁都能用 → 只是领域共识，重提

## 八、Pipeline S5b 如何调用本规范

S5b（专家心智提炼）是 pipeline 的独立 subagent，与 S5a（领域镜片）并行。

### 输入

- S2-mind 产出的心智候选：专家判断/直觉/反模式/决策启发式（来自 mind 源 + both 源的判断切片）
- sources 摘要：每篇来源的核心贡献与覆盖范围

### 处理流程

1. **候选识别**——扫描心智候选，提炼候选「专家镜片」表述（不是评论，是看世界的方式）
2. **三重验证**——对每个候选逐项跑验证1/2/3，收集跨语境证据：
   - 跨场景：该镜片在几个语境/来源里复现？列出对应 sources ID
   - 生成力：假设一个新问题，这个专家的镜片能预测其什么反应？
   - 排他性：换一位同领域专家，这条镜片还成立吗？
3. **分级**——按降级规则归类：3 重过=心智模型，1-2 重=启发式，排他性必过=反模式，0 重=丢弃
4. **矛盾标注**——把专家内分歧按三类（时间性/设定性/本质张力）挂到相应心智元素上
5. **诚实边界**——汇总覆盖盲区、单一来源结论、未表态方向
6. **输出结构化心智元素**——含 `verification` 字段 + `grounded_in` 结构占位

### 输出

- **3-7 个验证过的心智模型**：每个含三重验证证据、专家独特性标注、grounded_in 结构
- **5-10 条启发式**：从只过 1-2 重的候选中降级保留的实操规则
- **若干反模式**：专家明确反对的方向（排他性必过）
- **诚实边界段落**：未覆盖方向 + 单一来源结论 + 未表态开放问题 + 来源清单

### 一次完整运行样例

输入：LeCun 专家库 30 个心智候选 + 3 篇来源摘要（JEPA 论文 + 访谈 + 社媒）。

- **候选识别**：扫到 8 个候选镜片（能量优化世界观、反对纯自回归、规划需要 world model、反对用 scaling law 解决一切、物理模拟的重要性……）
- **三重验证 + 分级后**：
  - 心智模型（3-7 个）：智能 = 能量优化 / 反对纯自回归 LLM / 规划需要 world model（共 3 个，全过 3 重）
  - 启发式（5-10 条）：对新架构先问能量函数、优先 latent space 而非 pixel space、不相信纯数据 scaling（过 1-2 重降级而来）
  - 反模式：纯自回归 LLM 作为 AGI 路线（排他性必过）
  - 丢弃：「相关不等于因果」（0 重，领域共识）
- **诚实边界**：标注未覆盖的方向（如对扩散模型做世界模型的具体技术评论）+ 单一来源结论 + 未表态开放问题

自检条数：心智模型 <3 → 回去扩大候选来源（多半材料太少）；>7 → 回去合并/丢弃（没收敛）。

## 九、与 S5a（领域镜片）的边界澄清

| 维度 | S5a（领域镜片）| S5b（专家心智）|
|------|----------------|----------------|
| **锚点** | 「换个领域就不这么想」 | 「换个人就不这么想」 |
| **验证标准** | 排他性=领域特有 | 排他性=专家特有 |
| **输出类型** | 领域镜片（写入 wiki/overview）| 心智模型/启发式/反模式（写入 expert-mind/mental-models.md）|
| **矛盾处理** | 领域内不同流派的分歧 | 专家内不同时间的分歧 |
| **诚实边界** | 未覆盖的领域子方向 | 未覆盖的专家观点方向 |

两者是**并行独立 subagent**，可同时运行。S5c 在两者完成后，做「专家心智 + 知识节点」的紧耦合融合。
